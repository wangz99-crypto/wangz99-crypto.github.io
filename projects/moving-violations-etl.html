☁️ Automated Cloud ETL Pipeline

AWS Lambda · EventBridge · RDS (MySQL) · Secrets Manager · Relational Modeling

Project Summary

I built a cloud-hosted ETL pipeline that loads DC moving violations and daily weather into an AWS RDS (MySQL) database. The pipeline supports automated daily runs, secure secrets handling, and idempotent incremental updates (safe to re-run without duplicates).

Data Platform
  • Relational design: Weather and violations tables joined by date for analytics-ready queries
  • Automation: EventBridge triggers Lambda for scheduled ingestion
  • Reliability: Incremental logic inserts 0 rows when no new data is available (idempotent)
  • Security: DB credentials stored in AWS Secrets Manager (no hard-coded secrets)

How the Pipeline Works

This system follows a production-style pattern: define a clean relational schema, establish secure DB access, and automate incremental ingestion on a schedule.

Design

Ingestion

  • Fetch violations and weather data
  • Transform to a consistent schema
  • Insert into MySQL tables

Incremental + Idempotent

  • Compute a date window for new records
  • Use keys/logic to avoid duplicates
  • Safe to re-run (“0 rows inserted” when unchanged)

Security + Ops

  • DB credentials stored in Secrets Manager
  • RDS access controlled via VPC + security groups
  • EventBridge schedule triggers Lambda runs

What I Would Add Next

If this were deployed as a public-facing service, I would expose curated query endpoints (no raw SQL input), add authentication + rate limiting, and include data quality checks.

Next
  • API layer (curated endpoints) instead of direct DB access
